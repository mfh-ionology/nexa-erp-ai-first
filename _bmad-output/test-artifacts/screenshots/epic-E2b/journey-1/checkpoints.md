# Visual Checkpoint Manifest — Journey #1: Admin Login and Permission-Driven Sidebar

## Checkpoint 1: Login Page Loaded
- **When**: After navigating to /login (Step 1)
- **Screenshot file**: step-1-login-page.png
- **What to look for**: Login form visible with email input field, password input field, and a "Sign In" button. Clean layout, no error messages displayed.

## Checkpoint 2: Dashboard After Login
- **When**: After clicking Sign In with valid admin credentials (Step 3)
- **Screenshot file**: step-3-dashboard-after-login.png
- **What to look for**: Dashboard/main page loaded. App shell visible with a sidebar. The sidebar should show the System module section expanded (or expandable) containing links for: Users, Company Profile, Resource Registry, Access Groups. No error toasts or login failure messages.

## Checkpoint 3: Sidebar System Items Verified
- **When**: After verifying all sidebar items are present (Step 5)
- **Screenshot file**: step-5-sidebar-system-items.png
- **What to look for**: All four System module sidebar items clearly visible: "Users", "Company Profile", "Resource Registry", "Access Groups". These represent the FULL_ACCESS admin's permission-driven navigation.
