# Journey 28: Platform Intelligence Dashboard — Responsive Layout
## Visual Checkpoint Manifest

### Checkpoint 1: Tablet Layout (768px viewport)
- **When**: After navigating to /intelligence at 768x1024 viewport
- **Screenshot file**: step-1-tablet-layout.png
- **What to look for**:
  - KPI cards arranged in 2-column grid (not 4-column)
  - Dashboard sections stacked vertically (single column)
  - Sidebar collapsed or hamburger menu visible
  - No horizontal overflow on the page body
  - All content readable without horizontal scrolling

### Checkpoint 2: Tablet Scroll (768px viewport)
- **When**: After scrolling to see more sections at tablet viewport
- **Screenshot file**: step-2-tablet-sections-scroll.png
- **What to look for**:
  - Skill effectiveness table present and horizontally scrollable
  - Lower sections visible and stacked in single column
  - Content remains readable

### Checkpoint 3: Mobile Layout (375px viewport)
- **When**: After switching to 375x812 mobile viewport
- **Screenshot file**: step-3-mobile-layout.png
- **What to look for**:
  - KPI cards stacked vertically (single column)
  - Sections rendered as collapsible accordions
  - Summary Panel expanded by default, other sections collapsed
  - Table data shown in card format instead of table rows
  - No content overflow or broken layout
  - Floating "Publish Knowledge" button visible at bottom-right
  - All text readable at mobile width

### Checkpoint 4: Mobile Accordion Interaction
- **When**: After clicking an accordion section to expand/collapse on mobile
- **Screenshot file**: step-4-mobile-accordion-interaction.png
- **What to look for**:
  - Accordion section expanded correctly showing content
  - Content readable within expanded section
  - Layout remains intact, no overflow
