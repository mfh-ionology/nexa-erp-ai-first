# Visual Checkpoints — Journey 22: Platform Dashboard — Feature Gaps, Workflow Opportunities, Default Optimisation

## Checkpoint 1: Feature Gaps Section
- **When**: After Step 1–2 — navigate to /intelligence and scroll to Feature Gaps section
- **Screenshot file**: `step-2-feature-gaps-section.png`
- **What to look for**: Feature Gaps section card with header, insight entries showing title, module category, tenant count, frequency, severity badges (HIGH=red, MEDIUM=amber, LOW=green), possible bar chart visualization

## Checkpoint 2: Feature Gap Status Changed to REVIEWED
- **When**: After Step 3 — click status dropdown on a feature gap insight and select REVIEWED
- **Screenshot file**: `step-3-feature-gap-status-reviewed.png`
- **What to look for**: Status badge shows REVIEWED with updated colour/text, optimistic update reflected immediately

## Checkpoint 3: Workflow Opportunities Section
- **When**: After Step 4 — verify Workflow Opportunities section
- **Screenshot file**: `step-4-workflow-opportunities-section.png`
- **What to look for**: Workflow Opportunities section with entries showing pattern description, tenant count/percentage, action dropdown per entry (Review, Dismiss, Create Automation Suggestion)

## Checkpoint 4: Default Optimisation Section
- **When**: After Step 5 — verify Default Optimisation section
- **Screenshot file**: `step-5-default-optimisation-section.png`
- **What to look for**: Default Optimisation section with adoption percentage progress bars (e.g. 72% filled), tenant counts, grouped by module, "Make Default" button visible

## Checkpoint 5: Make Default Action Completed

- **Screenshot file**: `step-6-make-default-actioned.png`
