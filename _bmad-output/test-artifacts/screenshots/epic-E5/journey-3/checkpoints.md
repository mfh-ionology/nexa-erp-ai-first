# Visual Checkpoint Manifest — Journey #3: Send First Chat Message and Receive Streaming Response

## Checkpoint 1: User Message Sent with Typing Indicator
- **When**: After step 3 — user clicks Send button after typing "What is the current status of my company?"
- **Screenshot file**: `step-3-user-message-sent-typing-indicator.png`
- **What to look for**:
  - User message bubble visible on the right side of the conversation area with distinct user styling (e.g., purple/blue background)
  - Message text reads "What is the current status of my company?"
  - Below the user message, a typing indicator or loading animation is visible (dots, shimmer, or spinner) indicating the AI is generating a response
  - Chat input field is cleared and ready for the next message
  - Co-Pilot drawer is open and properly laid out

## Checkpoint 2: Completed AI Response in Conversation
- **When**: After step 5 — AI streaming response completes
- **Screenshot file**: `step-5-ai-response-complete.png`
- **What to look for**:
  - Full conversation visible: user message on the right, AI response on the left with grey/distinct AI styling
  - No typing indicator — streaming has completed
  - AI response is a complete, well-formed message (not truncated or empty)
  - The response is contextual — references the company or provides relevant information
  - Conversation area is scrolled to show the most recent messages
  - Chat selector dropdown should now show this conversation with an auto-generated title derived from the first user message (e.g., "What is the current status...")
