# Journey 5: Mark All Notifications as Read — Visual Checkpoints

## Checkpoint 1: Bell badge with unread count
- **When**: After login and dashboard load (Step 2)
- **Screenshot file**: `step-2-bell-badge-unread-count.png`
- **What to look for**: Header toolbar visible with purple 'N' logo, search bar, and notification bell icon. The bell icon has a red badge showing a number greater than 0 (e.g., '3' or '5'). This confirms unread notifications exist before we mark them all as read.

## Checkpoint 2: All notifications marked as read
- **When**: After clicking 'Mark All Read' button (Step 4)
- **Screenshot file**: `step-4-all-marked-as-read.png`
- **What to look for**: Notification dropdown still open. All notification items now show as READ state — no blue unread dots visible, titles should be normal weight (not bold/semibold), text colours are muted. The 'Mark All Read' button should be hidden or disabled (no unread items remain). The bell icon badge should be gone (no red badge) or show 0.
