# Visual Checkpoints — Journey 12: Collapse and Expand Category Sections

## Checkpoint 1: Preferences page loaded with all categories expanded
- **When**: After navigating to /system/notification-preferences and page loads
- **Screenshot file**: step-1-all-categories-expanded.png
- **What to look for**: Multiple category sections visible, all expanded showing event rows with toggle switches. Category headers have chevron-down icons. Column headers (In-App, Email, Push) visible within each section.

## Checkpoint 2: First category collapsed
- **When**: After clicking the first category section header to collapse it
- **Screenshot file**: step-2-first-category-collapsed.png
- **What to look for**: First category section is collapsed — only the header bar is visible, no toggle rows beneath it. Other categories remain expanded with their toggle rows visible. The chevron icon on the collapsed section should have rotated (pointing right or up).

## Checkpoint 3: First category re-expanded
- **When**: After clicking the same category header again to re-expand it
- **Screenshot file**: step-3-first-category-re-expanded.png
- **What to look for**: First category section is expanded again showing its event rows with toggle switches. All sections are now expanded. State matches the initial load appearance.
