# Visual Checkpoints — Journey 2: Open Notification Dropdown

## Checkpoint 1: Dashboard loaded with bell icon visible
- **When**: After login and navigation to dashboard (before opening dropdown)
- **Screenshot file**: `step-1-dashboard-with-bell.png`
- **What to look for**: Header visible with purple 'N' logo, search bar, notification bell icon with red unread badge. Dashboard content loaded.

## Checkpoint 2: Notification dropdown open
- **When**: After clicking the notification bell icon (Step 2)
- **Screenshot file**: `step-2-notification-dropdown-open.png`
- **What to look for**: Popover panel (~380px wide) visible below the bell icon. 'Notifications' heading in header. 'Mark All Read' button visible. List of notification items each showing: title, body preview, relative timestamp, and coloured left border (red=URGENT, amber=HIGH, blue=NORMAL). Unread notifications have a blue dot indicator and slightly tinted `#faf9ff` background. 'View All' link in footer.

## Checkpoint 3: Priority indicators and unread dots detail
- **When**: After verifying notification items (Steps 5-7)
- **Screenshot file**: `step-7-priority-and-timestamps.png`
- **What to look for**: Notification items with visible coloured left borders indicating priority. Blue unread dots next to notification titles. Relative timestamps (e.g., '5m ago', '2h ago') visible on each item.
