# Visual Checkpoint Manifest — Journey 1: Navigate to Document Templates Page

## CP-1: Dashboard After Login
- **When**: After logging in and dashboard loads (Step 1)
- **Screenshot file**: `step-1-dashboard-loaded.png`
- **What to look for**: Dashboard visible with sidebar navigation on the left, user is authenticated, no login form visible. Sidebar should show navigation groups including Administration section.

## CP-2: Document Templates Page Layout
- **When**: After navigating to /settings/document-templates (Step 3)
- **Screenshot file**: `step-3-document-templates-page.png`
- **What to look for**: T7 Settings layout visible with breadcrumb showing 'System > Document Templates'. Page title 'Document Templates' visible. Purple 'Add Template' button in the action area. Concept D background (#f4f2ff). Document type filter dropdown and search input in filter bar.

## CP-3: Template List with Grouped Content
- **When**: After verifying all page elements are present (Step 8)
- **Screenshot file**: `step-8-template-groups-and-cards.png`
- **What to look for**: Template cards grouped by document type in collapsible accordion sections. Each card shows template name, green 'Active' badge, purple 'Default' badge where applicable, page size info, and version count. At least 5 document type sections should be visible (first 5 expanded by default). Accordion headers show document type labels with count badges in purple (#6d28d9) styling.
