# Visual Checkpoint Manifest — Journey 7: Filter Audit Log by User ID

## Journey Summary
Admin user logs in, navigates to the Audit Log page, applies a User ID filter to show only records generated by that user, and verifies the filtered results.

## Checkpoints

### Checkpoint 1: Dashboard after login
- **When:** After Step 3 — clicking Sign In and successful redirect
- **Screenshot file:** step-3-dashboard-after-login.png
- **What to look for:** Dashboard loaded with app shell. Sidebar visible with System module section containing Audit Log link. User is authenticated as admin.

### Checkpoint 2: Audit Log page loaded
- **When:** After Step 4 — navigating to /system/audit-log
- **Screenshot file:** step-4-audit-log-page-loaded.png
- **What to look for:** Audit Log page with heading visible. Filter controls present including User filter (dropdown/combobox or text input for userId). Data table showing audit records with columns: Timestamp, Entity Type, Entity ID, Action, User, AI Action. Records populated from prior activity.

### Checkpoint 3: Filtered results by User ID
- **When:** After Step 6 — clicking Apply Filters with User filter set to admin user
- **Screenshot file:** step-6-filtered-by-user-id.png
- **What to look for:** Audit log table shows only records where the User column matches the admin user. All rows display the same userId/user name. Filter state is visible showing the active User filter. Pagination reflects the filtered count.
