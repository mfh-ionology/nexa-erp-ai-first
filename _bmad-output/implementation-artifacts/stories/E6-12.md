# Story E6-12: Configuration Export & Import UI

**Epic:** E6 — Web Frontend Shell + Mobile Scaffold
**Status:** done
**Dependencies:** E6.1 (React App Bootstrap), E6.2 (Navigation Shell — sidebar with system module navigation), E6.4 (ActionBar Component — overflow menu pattern), E2b.6 (Export/import backend API — `GET /system/company-profile/export-defaults`, `POST /system/company-profile/import-defaults`)
**Complexity:** Medium (9 tasks)

---

## User Story

As an administrator, I want to export my company's permission configuration as JSON and import it into other companies, so that I can standardise permissions across multiple companies or share best-practice configurations.

---

## Acceptance Criteria

### AC1: Export Config Overflow Menu Action
**GIVEN** an authenticated administrator navigates to `/system/companies`
**WHEN** they click the overflow menu (⋯ More) in the page action bar
**THEN** an "Export Config" option is visible in the Data section of the overflow menu
**AND** the action is only visible to users with ADMIN or SUPER_ADMIN role (BR-RBAC-008)

### AC2: Export Preview Dialog
**GIVEN** the administrator clicks "Export Config" from the overflow menu
**WHEN** the export dialog opens
**THEN** it displays a summary preview showing the count of items that will be exported: access groups, permissions, field overrides, VAT codes, payment terms, number series, currencies
**AND** a "Download JSON" primary action button is visible
**AND** a "Cancel" ghost button is visible
**AND** the dialog title is "Export Company Configuration"

### AC3: Export Download
**GIVEN** the export preview dialog is open
**WHEN** the administrator clicks "Download JSON"
**THEN** the system calls `GET /system/company-profile/export-defaults`
**AND** the response is downloaded as a JSON file named `company-defaults-{companySlug}.json`
**AND** a success toast confirms "Configuration exported successfully"
**AND** the dialog closes after successful download

### AC4: Export Loading State
**GIVEN** the export download is in progress
**WHEN** the API call is pending
**THEN** the "Download JSON" button shows a spinner and is disabled
**AND** the "Cancel" button remains active

### AC5: Import Config Overflow Menu Action
**GIVEN** an authenticated administrator navigates to `/system/companies`
**WHEN** they click the overflow menu (⋯ More) in the page action bar
**THEN** an "Import Config" option is visible in the Data section of the overflow menu
**AND** the action is only visible to users with ADMIN or SUPER_ADMIN role (BR-RBAC-008)

### AC6: Import Dialog — File Upload
**GIVEN** the administrator clicks "Import Config" from the overflow menu
**WHEN** the import dialog opens
**THEN** it displays: a file drop zone accepting `.json` files, a JSON textarea for pasting, a "Dry Run" checkbox (checked by default), and an "Import" primary action button (disabled until file/JSON is provided)
**AND** the dialog title is "Import Company Configuration"
**AND** a "Cancel" ghost button is visible

### AC7: Import Dialog — File Selection
**GIVEN** the import dialog is open
**WHEN** the administrator drops a `.json` file or clicks to browse and selects a file
**THEN** the file name is displayed in the drop zone
**AND** the file contents are parsed and validated as JSON
**AND** if the file is not valid JSON, an inline error is shown: "Invalid JSON file. Please select a valid configuration file."
**AND** the Import button becomes enabled when valid JSON is loaded

### AC8: Import Dialog — JSON Textarea
**GIVEN** the import dialog is open
**WHEN** the administrator pastes JSON into the textarea instead of uploading a file
**THEN** the JSON is validated on blur
**AND** if the JSON is invalid, an inline error is shown below the textarea: "Invalid JSON format"
**AND** the Import button becomes enabled when valid JSON is entered
**AND** using the textarea clears any previously uploaded file, and vice versa

### AC9: Import Dry Run Mode
**GIVEN** the administrator has provided valid JSON (via file or textarea)
**WHEN** the "Dry Run" checkbox is checked and they click "Import"
**THEN** the system calls `POST /system/company-profile/import-defaults` with `{ data: <parsedJSON>, dryRun: true }`
**AND** the response with `status: 'DRY_RUN'` is displayed in a results panel within the dialog
**AND** the results panel shows a summary table: resources created/updated, access groups created/updated, permissions set, field overrides set, VAT codes created/updated, payment terms created/updated, number series created/updated, currencies created/updated
**AND** any warnings from the response are displayed below the summary
**AND** the dialog remains open with an "Apply" button to proceed with actual import, and a "Close" button to dismiss

### AC10: Import Apply Mode
**GIVEN** a dry run has completed successfully
**WHEN** the administrator clicks "Apply" (or unchecks dry run and clicks "Import")
**THEN** the system calls `POST /system/company-profile/import-defaults` with `{ data: <parsedJSON>, dryRun: false }`
**AND** the response with `status: 'APPLIED'` is displayed in the results panel
**AND** a success toast confirms "Configuration imported successfully"
**AND** the results panel shows final counts of created/updated records
**AND** the dialog shows a "Done" button to close

### AC11: Import Error Handling
**GIVEN** the import request fails
**WHEN** the response is `400 VALIDATION_ERROR` (invalid JSON structure or missing required fields)
**THEN** an error message is displayed in the dialog: the server's error message
**AND** the dialog remains open for the administrator to correct and retry
**WHEN** the response is `400 UNSUPPORTED_VERSION` (file version not supported)
**THEN** an error message is displayed: "This configuration file version is not supported. Please export a new configuration from the source company."
**AND** the dialog remains open

### AC12: Import Loading State
**GIVEN** the import request is in progress
**WHEN** the API call is pending
**THEN** the "Import" or "Apply" button shows a spinner and is disabled
**AND** the file upload area and textarea are disabled
**AND** the "Cancel" button remains active

### AC13: Company Profile Page Shell
**GIVEN** an authenticated administrator
**WHEN** they navigate to `/system/companies` via the sidebar
**THEN** a Company Profile page renders with: breadcrumb (System > Company Profile), page title showing the current company name, and an action bar with an overflow menu
**AND** the overflow menu contains "Export Config" and "Import Config" in the Data section

### AC14: Accessibility
**GIVEN** the export and import dialogs
**WHEN** using keyboard navigation
**THEN** all interactive elements (buttons, file input, textarea, checkbox, close button) are keyboard-navigable with visible focus indicators (NFR28)
**AND** the dialogs follow WAI-ARIA dialog pattern with `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to the dialog title
**AND** file drop zone has `aria-label="Upload configuration file"` and responds to Enter/Space to open file picker
**AND** success/error toasts use `role="status"` for screen reader announcement
**AND** colour contrast meets WCAG 2.1 AA (NFR27)

### Task 2: Create Export/Import API Hooks
**AC:** #3, #4, #9, #10, #11, #12
**Files:**
- `apps/web/src/features/admin/company-config/api/use-company-config-mutations.ts` (create)

**Subtasks:**
- [x] 2.1 Create `useExportDefaults()` mutation hook:
  - Calls `apiGet<ExportDefaultsResponse>('/system/company-profile/export-defaults')`
  - On success: triggers file download (construct a Blob from the JSON response, create a temporary `<a>` element with `download` attribute set to `company-defaults-{companySlug}.json`, click it)
  - Show success toast: `t('companyConfig.toast.exportSuccess')`
  - On error: show error toast: `t('companyConfig.error.exportFailed')`
- [x] 2.2 Create `useImportDefaults()` mutation hook:
  - Calls `apiPost<ImportDefaultsResponse>('/system/company-profile/import-defaults', body)` where body is `ImportDefaultsRequest`
  - Returns `ImportDefaultsResponse` on success
  - Does NOT show toast on success (caller handles display since results are shown in dialog)
  - On 400 `VALIDATION_ERROR`: extract error message from response, throw to caller (do NOT show generic toast — caller displays contextual error in dialog)
  - On 400 `UNSUPPORTED_VERSION`: extract error code, throw to caller
  - On unexpected error: show generic error toast via `t('errors:unexpected')`
- [x] 2.3 Import types from `./types` and use `apiGet`, `apiPost` from `@/lib/api-client`

**Implementation notes:**
- The export hook initiates a file download rather than returning data for display. Use the standard Blob + `URL.createObjectURL` + `<a download>` pattern for browser-initiated downloads.
- The import hook does NOT auto-show toasts for success because the dialog needs to display the results summary inline. The calling component handles success display.
- Follow the same mutation pattern from `features/admin/access-groups/api/use-access-group-mutations.ts`

- The "Apply" button only appears after a successful dry-run. If the user unchecks dry-run and clicks "Import", it goes directly to APPLIED mode.
- Use `React.useState` for dialog phase tracking, parsed JSON storage, and results storage. Do NOT use TanStack Query for this — mutations with local state are the right pattern.

## Dev Notes

1. **Backend Already Complete**: The backend API for export/import was built and tested in E2b.6 (all tests passed). This story is frontend-only — no backend changes needed.

2. **File Download Pattern**: For the export, use the standard browser download pattern:
   ```typescript
   const blob = new Blob([JSON.stringify(data, null, 2)], { type: 'application/json' });
   const url = URL.createObjectURL(blob);
   const a = document.createElement('a');
   a.href = url;
   a.download = `company-defaults-${companySlug}.json`;
   a.click();
   URL.revokeObjectURL(url);
   ```

3. **File Upload Pattern**: For the import, read the file client-side and send as inline JSON. Do NOT use multipart upload:
   ```typescript
   const reader = new FileReader();
   reader.onload = (e) => {
     try {
       const data = JSON.parse(e.target?.result as string);
       setParsedData(data);
     } catch {
       setFileError(t('companyConfig.import.invalidFile'));
     }
   };
   reader.readAsText(file);
   ```

4. **Dialog State Machine**: The import dialog has three phases: `input` → `dryRunResults` → `appliedResults`. The input phase shows the file/JSON upload controls. The results phases show the summary table. Transitioning between phases replaces the dialog body content but preserves the parsed JSON data in state.

5. **Minimal Company Profile Page**: The Company Profile page created in this story is intentionally minimal — it's a shell to host the overflow menu with export/import actions. The full company settings functionality (editing company name, address, logo, etc.) will be built in a later story/epic. Do NOT over-build this page.

6. **No Company Profile Query**: This story does NOT create a `useCompanyProfile()` query hook or fetch company details from the API. The company name shown in the page title comes from `useAuthStore().activeCompany.name`, which is already available from the auth context.

7. **Route Registration**: The route file at `routes/_authenticated/system/companies.tsx` must be created before running `npx tsr generate` to register it in the route tree. The nav config already has the `system.companies` menu item pointing to `/system/companies`.

8. **Permission Gating**: The overflow menu actions (Export Config, Import Config) should check `useAuthStore().role` for `ADMIN` or `SUPER_ADMIN` before rendering. The API endpoints also enforce ADMIN auth, so this is defense-in-depth.
