# Story E5d-6: Platform Intelligence Dashboard (Vendor)

**Epic:** E5d — AI Knowledge Evolution & Cross-Tenant Intelligence
**Status:** done
**Dependencies:** E5d-3 (Cross-Tenant Intelligence Pipeline — done), E5d-4 (Platform Knowledge Distribution — backlog, knowledge publish endpoints needed), E3b (Platform API — done)
**Complexity:** X-Large (11 tasks)
**FRs:** FR6 (AI-powered analytics) | **NFRs:** NFR2 (CRUD <500ms), NFR27 (WCAG 2.1 AA), NFR28 (keyboard navigable), NFR29 (contrast 4.5:1)

---

## User Story

As a **platform vendor**,
I want a dashboard in the Platform Admin Portal showing cross-tenant AI intelligence — feature gaps, workflow opportunities, skill effectiveness, correction patterns, and industry breakdowns — so that I can make data-driven product decisions, improve the base AI for all tenants, and publish curated knowledge to the right audiences.

---

## Acceptance Criteria

### AC1: Platform Admin App Shell & Routing
**GIVEN** the `apps/platform-admin/` application (currently a stub)
**WHEN** the platform admin navigates to the app
**THEN** the Platform Admin Portal renders as a standalone Vite + React + TanStack Router application on port 5112 with:
  - Sidebar navigation with "PLATFORM ADMIN" branding and purple "N" logo
  - Navigation items: `Dashboard` (home), `AI Intelligence` (this story), `Knowledge Management` (future)
  - Authentication against the Platform API's JWT system (reusing `requirePlatformRole` from E3b)
  - Layout matching the Concept D design system: dark sidebar, `#f4f2ff` content background, Plus Jakarta Sans headings
  - The app's entry point configured in the monorepo's `turbo.json` and `package.json` dev scripts
**AND** the `AI Intelligence` route is at `/intelligence` within the platform-admin app
**AND** the route is protected — only `PLATFORM_ADMIN` and `PLATFORM_VIEWER` roles can access

### AC2: Dashboard Overview Panel (Summary KPIs)
**GIVEN** the AI Intelligence page at route `/intelligence`
**WHEN** a platform admin navigates to it
**THEN** the top of the page displays a KPI panel with 4 metric cards:
  - **Contributing Tenants** — total tenants sharing anonymised data (from `GET /admin/intelligence/summary`)
  - **Knowledge Articles** — total knowledge articles across all tenants
  - **Total Corrections** — total AI corrections logged across tenants
  - **AI Success Rate** — overall success rate as a percentage with trend indicator (↑/↓/→)
**AND** each card uses the Concept D card style: 12px radius, custom shadow, `JetBrains Mono` for numbers
**AND** the summary data loads via `GET /admin/intelligence/summary` with a loading skeleton during fetch
**AND** the panel has a "Last aggregated: {date}" timestamp and a "Refresh" action to trigger re-aggregation

### AC3: Feature Gaps Section
**GIVEN** the AI Intelligence page
**WHEN** scrolling to the "Feature Gaps" section
**THEN** a section displays the top 10 AI queries that fail (no matching skill or low confidence):
  - Each entry shows: query description (anonymised), module category, tenant count, frequency count
  - Entries are sorted by frequency descending (most impactful gaps first)
  - A bar chart visualisation shows feature gaps grouped by module (AR, Finance, Sales, etc.)
  - Each gap entry has a severity badge: HIGH (red) if >10 tenants, MEDIUM (amber) if 5-10, LOW (green) if <5
**AND** data comes from `GET /admin/intelligence/insights` filtered by `insightType=FEATURE_GAP`
**AND** the section has a "View All" link that expands to show all feature gap insights beyond the top 10
**AND** each insight can be marked as REVIEWED or ACTIONED via inline status update (`PATCH /admin/intelligence/insights/:id`)

### AC4: Workflow Opportunities Section
**GIVEN** the AI Intelligence page
**WHEN** scrolling to the "Workflow Opportunities" section
**THEN** a section displays detected manual patterns across tenants:
  - Each entry shows: pattern description, tenant count, frequency, automation candidate status
  - Example: "40% of tenants export aging report then email it — automation candidate"
  - Entries are sorted by tenant count descending
  - Each entry has an "Action" dropdown: Review, Dismiss, Create Automation Suggestion
**AND** data comes from `GET /admin/intelligence/insights` filtered by `insightType=WORKFLOW_OPPORTUNITY`
**AND** insights can be updated to REVIEWED/ACTIONED/DISMISSED via `PATCH /admin/intelligence/insights/:id`

### AC5: Default Optimisation Section
**GIVEN** the AI Intelligence page
**WHEN** scrolling to the "Default Optimisation" section
**THEN** a section displays configurations that >60% of tenants create manually:
  - Each entry shows: configuration description, adoption percentage, tenant count
  - Example: "72% of construction tenants create 'Retention Invoices' view → make it a default"
  - Entries are grouped by module and sorted by adoption percentage descending
  - Each entry has a "Make Default" action button that transitions the insight to ACTIONED status
**AND** data comes from `GET /admin/intelligence/insights` filtered by `insightType=DEFAULT_CANDIDATE`

### AC6: Skill Effectiveness Table
**GIVEN** the AI Intelligence page
**WHEN** scrolling to the "Skill Effectiveness" section
**THEN** a sortable, filterable table displays all skills with cross-tenant metrics:
  - Columns: Skill Name, Module, Avg Success Rate (%), Avg Correction Rate (%), Usage Count, Tenant Count, Confidence, Trend
  - **Trend** column shows an icon: ↑ green (IMPROVING), → amber (STABLE), ↓ red (DECLINING)
  - **Success Rate** cells are colour-coded: green (>80%), amber (50-80%), red (<50%)
  - A module filter dropdown allows filtering skills by module (AR, Finance, Sales, etc.)
  - Table supports sorting by any column (click header to sort)
  - Skills with correction rate >30% are highlighted with a warning indicator
**AND** data comes from `GET /admin/intelligence/skill-effectiveness`
**AND** the table uses cursor pagination with 20 items per page
**AND** numbers use `JetBrains Mono` font and percentage values show 1 decimal place

### AC7: Industry Breakdown Section
**GIVEN** the AI Intelligence page
**WHEN** scrolling to the "Industry Breakdown" section
**THEN** a section displays patterns grouped by tenant industry:
  - An industry selector dropdown allows choosing: Construction, Retail, Manufacturing, Professional Services, All
  - When an industry is selected, the section shows:
    - Top query categories for that industry
    - Most-used skills for that industry
    - Common correction types for that industry
    - Tenant count in that industry
  - A comparison mode allows selecting two industries to compare patterns side-by-side
**AND** data comes from `GET /admin/intelligence/patterns` filtered by `industry` parameter
**AND** the industry selector updates the displayed data without a full page reload

### AC8: Correction Patterns Section
**GIVEN** the AI Intelligence page
**WHEN** scrolling to the "Correction Patterns" section
**THEN** a section displays the most common AI mistakes across tenants:
  - Corrections are grouped by category: TERMINOLOGY, PROCESS, DATA, PREFERENCE
  - Each group shows: correction type, related skill (if any), occurrence count, tenant count, common correction text (anonymised)
  - A skill filter allows narrowing corrections to a specific skill
  - Each correction pattern has a "Create Knowledge Article" action that navigates to the platform knowledge article creation form pre-filled with the correction context
**AND** data comes from `GET /admin/intelligence/corrections`
**AND** the section supports cursor pagination within each category group

### AC9: Publish Knowledge Workflow
**GIVEN** the AI Intelligence page
**WHEN** a platform admin clicks "Publish Knowledge" (from any section's action button, or from a dedicated "Publish" action in the page action bar)
**THEN** a side panel or modal form opens with:
  - Title field (required, max 500 chars)
  - Content field (rich text / markdown editor, required)
  - Category selector: BEST_PRACTICE, HELP, DEFAULT_CONFIG, SKILL_UPDATE
  - Target Industries multi-select (empty = all industries)
  - Target Plan Tiers multi-select (empty = all tiers)
  - Preview panel showing the article as tenants will see it
  - "Save as Draft" and "Publish" buttons
**AND** "Save as Draft" calls `POST /admin/intelligence/knowledge` with the form data
**AND** "Publish" calls `POST /admin/intelligence/knowledge` then `POST /admin/intelligence/knowledge/:id/publish`
**AND** when triggered from a correction pattern or insight, the form is pre-filled with relevant context
**AND** the form validates all required fields before submission
**AND** success shows a toast notification with the article's distribution reach (eligible tenant count)

### AC10: Responsive Layout & Accessibility
**GIVEN** the AI Intelligence page
**WHEN** viewed on different screen sizes
**THEN** the layout adapts:
  - **Desktop (≥1280px):** Full dashboard with KPI grid (4 columns), side-by-side sections, expanded table
  - **Tablet (768–1279px):** KPI grid (2 columns), stacked sections, horizontally scrollable table
  - **Mobile (<768px):** KPI cards stacked vertically, sections as collapsible accordions, table rows as cards
**AND** all interactive elements are keyboard navigable (Tab, Enter, Escape, Arrow keys in tables)
**AND** all icons have `aria-label` attributes
**AND** colour-coded indicators (success rate, trend) have text alternatives (not colour-only)
**AND** contrast ratios meet WCAG 2.1 AA (≥4.5:1 for text, ≥3:1 for UI components)
**AND** the page uses `prefers-reduced-motion` to disable animations when requested

### AC11: Data Refresh & Loading States
**GIVEN** the AI Intelligence page
**WHEN** data is being fetched
**THEN** each section shows a skeleton loader matching the final layout shape
**AND** failed API calls show an error state with a "Retry" button (per-section, not whole page)
**AND** a "Last Updated" timestamp shows when data was last aggregated
**AND** a manual "Run Aggregation" button (PLATFORM_ADMIN only, not PLATFORM_VIEWER) triggers `POST /admin/intelligence/aggregate` with a confirmation dialog
**AND** a manual "Generate Insights" button (PLATFORM_ADMIN only) triggers `POST /admin/intelligence/generate-insights` with a confirmation dialog
**AND** after triggering aggregation or insights generation, the page polls for completion and auto-refreshes affected sections

## Architecture Decision: Platform Admin as Separate App

The Platform Admin Portal is a **separate Vite + React application** (`apps/platform-admin/`), not a section within the tenant web app (`apps/web/`). This is mandated by the architecture document (§2.31) for several reasons:

1. **Security isolation** — Platform admins access cross-tenant data; tenant users must never see platform routes
2. **Different auth model** — Platform JWT vs tenant JWT, different role hierarchy
3. **Independent deployment** — Platform admin can be deployed to a different domain/subdomain
4. **Simpler code splitting** — No conditional loading of platform code in tenant bundles

The app reuses `@nexa/ui` for shared components and Tailwind theme but has its own routing, auth, and API client.

## Dev Notes

- **Port 5112**: Assigned per the 5100–5200 port range rule. API=5100, Platform API=5101, Web=5110, Platform Admin=5112.
- **No charting library for MVP**: Use CSS-only bar charts (Tailwind `w-[{percent}%]` bars) or simple SVG. If a charting library is needed later, consider `recharts` (already used if present) or `@nivo/bar`.
- **Comparison mode (AC7)**: This is a stretch feature. If time is tight, implement the single-industry view first and defer comparison to a follow-up.
- **Pre-fill pattern (AC9)**: Use URL search params or React state to pass context from insight/correction to the publish panel. Prefer state (simpler) since it's within the same page.
- **Cursor pagination**: The Platform API returns `{ data: T[], nextCursor: string | null, hasMore: boolean }`. The frontend should use `useInfiniteQuery` from TanStack Query for paginated lists or manual cursor state for tables.
- **Stale time**: Summary data has 5-minute stale time (it's a daily snapshot, not real-time). Insights and patterns have 1-minute stale time.
- **PLATFORM_VIEWER role**: Can view all intelligence data but cannot trigger aggregation, generate insights, update insight status, or publish knowledge. Guard these mutations in the UI by checking the user's role.
- **Privacy enforcement**: The dashboard ONLY shows anonymised aggregates. No tenant names, company names, user names, or individual data points should ever appear in the UI. Industry labels and plan tiers are the finest granularity.
