# Story E13b.4: AI Usage Dashboard

Status: done

## Story

As a **platform administrator**,
I want AI usage charts (daily/weekly/monthly), quota alerts, spike detection, and per-tenant usage breakdowns,
so that I can monitor AI costs and identify anomalies.

## Acceptance Criteria

1. GIVEN the AI Usage page WHEN it loads THEN it shows cross-tenant usage summary: total tokens today, this month, cost estimate, and trend chart
2. GIVEN per-tenant AI usage WHEN the admin drills into a tenant THEN they see usage by feature (chat, document processing, forecasting), by provider (Anthropic, OpenAI, etc.), daily trend (30-day), quota progress bar, and BYOK vs vendor key breakdown
3. GIVEN quota alerts WHEN a tenant crosses the soft limit (80%) THEN an alert appears in the alerts list with tenant name, usage percentage, and timestamp
4. GIVEN spike detection WHEN a tenant's daily usage exceeds 3x their 7-day rolling average THEN an anomaly alert is flagged for investigation
5. GIVEN the AI Usage page WHEN the admin clicks "Export CSV" THEN a CSV file downloads with per-tenant, per-day usage data
6. GIVEN the Platform Admin AI settings page WHEN the admin views provider configuration THEN they see vendor-level API keys per provider (masked), active/inactive status, and can update keys
7. GIVEN an Enterprise tier tenant WHEN the admin views the tenant's AI configuration THEN they see any BYOK API keys configured per provider (masked), active/inactive status, and usage split between BYOK and vendor keys

### Architecture Context

This story delivers the **AI Usage Dashboard** for the Platform Admin Portal. It spans both backend (Platform API endpoints) and frontend (Platform Admin React app). The data source is the `TenantAiUsage` table — an append-only log of every AI call made through the AI Gateway, written by `POST /platform/tenants/:tenantId/ai/record` (implemented in E3b).

| Concern | Details |
|---------|---------|
| Database | Platform PostgreSQL (NOT tenant DB) |
| API target | `apps/platform-api/` (port 5101) |
| Frontend app | `apps/platform-admin/` (port 5112) |
| Data source | `tenant_ai_usage` table (append-only, written by AI Gateway) |
| Quota tracking | `tenant_ai_quotas` table (rolling period totals) |
| Alert storage | NEW `platform_ai_alerts` table (created in this story) |
| Provider keys | `tenant_provider_credentials` (BYOK), NEW `vendor_provider_credentials` (vendor) |

### Business Rules

| Rule | Description | Enforcement |
|------|-------------|-------------|
| BR-PLT-007 | All AI calls must route through the AI Gateway | HARD — architecture constraint (already enforced) |
| BR-PLT-008 | AI Gateway checks quota before every call | CONFIGURABLE — per-plan hard limit flag |
| BR-PLT-009 | AI usage records are durable (zero-loss) | HARD — local queue + retry |
| BR-PLT-010 | Quota alerts at configurable thresholds (50%, 80%, 100%) | CONFIGURABLE — per-tenant soft/hard limit |
| BR-PLT-011 | Spike detection: daily > 3x rolling 7-day average | INFORMATIONAL — background job |

### State Machine: AI Quota (Runtime)

Not a persisted state machine but runtime calculation based on `tokensUsed / tokenAllowance`:

| Threshold | State | ERP Behaviour |
|-----------|-------|---------------|
| 0-49% | NORMAL | AI calls proceed normally |
| 50% (configurable) | ALERT_50 | Platform admin notified. No tenant impact |
| 80% (softLimitPct) | SOFT_LIMIT | `tenant.quota_warning` event. ERP shows banner |
| 100% (hardLimitPct) | HARD_LIMIT | If aiHardLimit=true: AI calls blocked. If false: overage logged |
| >100% + spike | ANOMALY | Daily > 3x 7-day avg: platform admin alert |

### Platform Events

| Event | Trigger | Consumers |
|-------|---------|-----------|
| `tenant.quota_warning` | Usage crosses softLimitPct | ERP webhook (show banner), Platform notification |
| `tenant.quota_exceeded` | Usage crosses hardLimitPct | ERP webhook (block AI), Platform alert |
