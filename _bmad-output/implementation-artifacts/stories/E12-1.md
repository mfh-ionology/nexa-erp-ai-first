**Epic:** E12 ŌĆö Document Templates & PDF
**Status:** done
**Dependencies:** E0 (Foundation ŌĆö Prisma, core infra), E1 (System Module ŌĆö CompanyProfile, User models), E10 (Email ŌĆö BullMQ infrastructure, email-send worker pattern)
**Complexity:** X-Large (10 tasks)

## User Story

As a **system**,
I want to compile Handlebars HTML templates and render them to PDF via Puppeteer, with variable injection and conditional sections, so that business documents (invoices, POs, statements) are generated as professional PDFs.

## Acceptance Criteria

### AC1: Handlebars Template Compilation
**GIVEN** a `DocumentTemplate` record with an `htmlTemplate` containing Handlebars syntax
**WHEN** the `TemplateCompilerService` compiles the template with a data context
**THEN** all Handlebars expressions are resolved:
  - Simple variables: `{{invoice.number}}`, `{{customer.name}}`, `{{company.vatNumber}}`
  - Nested access: `{{company.address.line1}}`
  - Each blocks: `{{#each lines}}...{{/each}}` ŌĆö iterates over line items
  - Conditional blocks: `{{#if showVatNumber}}...{{/if}}` ŌĆö includes/excludes sections
  - Else blocks: `{{#if showLogo}}...{{else}}...{{/if}}`
**AND** the compiled output is valid HTML ready for PDF rendering
**AND** missing variables resolve to empty string (no `undefined` in output)
**AND** compilation completes within 100ms for templates up to 50KB

### AC2: Handlebars Helper Functions
**GIVEN** the template contains helper function calls
**WHEN** compiled with a data context
**THEN** the following custom helpers are available:
  - `{{formatCurrency amount currency}}` ŌĆö formats `5000` as `┬Ż5,000.00` (using currency symbol and locale)
  - `{{formatDate date format}}` ŌĆö formats ISO date as `DD/MM/YYYY` (UK default) or custom format
  - `{{formatNumber value decimals}}` ŌĆö formats `1234.5` as `1,234.50` with locale grouping
  - `{{eq a b}}` ŌĆö equality comparison for conditionals: `{{#if (eq status "POSTED")}}...{{/if}}`
  - `{{gt a b}}` / `{{lt a b}}` ŌĆö numeric comparison helpers
  - `{{uppercase str}}` / `{{lowercase str}}` ŌĆö string transform helpers
  - `{{lineNumber @index}}` ŌĆö returns 1-based line number from 0-based index
**AND** currency formatting respects the document's currency code (GBP ŌåÆ `┬Ż`, USD ŌåÆ `$`, EUR ŌåÆ `Ōé¼`)
**AND** date formatting defaults to UK format (`DD/MM/YYYY`) unless overridden

### AC3: Version Selection Algorithm
**GIVEN** a `DocumentTemplate` with multiple `DocumentTemplateVersion` records
**WHEN** `selectTemplateVersion()` is called with a context (`languageCode`, `branchCode`, `numberSeriesId`, `accessGroup`, `customerGroupId`)
**THEN** the system scores each active version against the context:
  - `languageCode` match: +10 points; mismatch (non-null, different value): -20 points
  - `branchCode` match: +8 points; mismatch: -16 points
  - `numberSeriesId` match: +6 points
  - `accessGroup` match: +4 points
  - `customerGroupId` match: +2 points
  - Null criteria are wildcards (neither add nor subtract)
**AND** versions are sorted by match score (descending), then by `priority` (descending) as tiebreaker
**AND** the highest-scoring version is returned
**AND** if no version matches or all versions score negatively, the base template (no version overrides) is used
**AND** only active versions (`isActive = true`) are considered

### AC4: Data Context Loading
**GIVEN** a `documentType` and `recordId`
**WHEN** the `DocumentDataLoaderService` fetches the data context
**THEN** it returns a structured object containing:
  - **Company data**: name, legalName, address, vatNumber, companyNumber, bankName, bankSortCode, bankAccountNumber, logoUrl (from `CompanyProfile`)
  - **Document data**: number, date, dueDate, reference, notes, status, currency (from the source record)
  - **Counterparty data**: name, address, vatNumber, contactEmail (customer or supplier depending on documentType)
  - **Line items**: array of `{ description, quantity, unitPrice, vatRate, lineTotal, itemCode }` (from document lines)
  - **Totals**: subtotal, vatAmount (grouped by rate), total, amountDue (total - payments received)
  - **Metadata**: paymentTerms name, currency code/symbol, formatted dates
**AND** monetary values are formatted using `formatCurrency` for display but also available as raw `Decimal` for calculation
**AND** the data loader gracefully handles missing relations (e.g., no payment terms ŌåÆ empty string)
**AND** `companyId` scoping is enforced ŌĆö only records belonging to the authenticated user's company are loaded

### AC5: Puppeteer HTML-to-PDF Rendering
**GIVEN** compiled HTML content from the template compiler
**WHEN** `PdfGeneratorService.generatePdf()` is called
**THEN** it renders the HTML to a PDF using Puppeteer (headless Chrome) with:
  - Page size from template: `A4` (default), `A5`, `Letter`
  - Orientation from template: `portrait` (default), `landscape`
  - Margins from template: top/bottom (default 20mm), left/right (default 15mm)
  - Optional header HTML (rendered on every page)
  - Optional footer HTML with page numbers: `Page {{pageNumber}} of {{totalPages}}`
**AND** the PDF respects CSS `page-break-before`, `page-break-after`, and `page-break-inside: avoid` rules
**AND** the generated PDF is returned as a `Buffer`
**AND** PDF generation completes within 5 seconds for documents up to 50 pages (NFR3)
**AND** the Puppeteer browser instance is reused across requests (connection pooling, not launched per-request)
**AND** if Puppeteer fails (crash, timeout), the error is caught and a meaningful error is returned (no orphaned browser processes)

### AC6: Document Generation Endpoint
**GIVEN** an authenticated user with STAFF role or above
**WHEN** `POST /api/v1/documents/generate` is called with `{ documentType, recordId, outputFormat? }`
**THEN** the system:
  1. Finds the active default template for the given `documentType`
  2. Selects the best template version based on document/user context (AC3)
  3. Loads the data context for the record (AC4)
  4. Compiles the template with the data context (AC1, AC2)
  5. Merges version overrides (htmlOverride, cssOverride, headerOverride, footerOverride) with base template
  6. Renders the compiled HTML to PDF (AC5)
  7. Returns the PDF as:
     - `outputFormat = "inline"` (default): `Content-Type: application/pdf`, `Content-Disposition: inline` (browser preview)
     - `outputFormat = "attachment"`: `Content-Type: application/pdf`, `Content-Disposition: attachment; filename="INV-00042.pdf"`
**AND** the response includes headers: `X-Document-Type`, `X-Template-Id`, `X-Template-Version-Id`
**AND** if no template exists for the documentType, returns 404 with error `TEMPLATE_NOT_FOUND`
**AND** if the recordId doesn't exist or belongs to a different company, returns 404 with error `RECORD_NOT_FOUND`
**AND** `companyId` scoping is enforced on both template and record lookups

### AC7: Batch Document Generation
**GIVEN** an authenticated user with MANAGER role or above
**WHEN** `POST /api/v1/documents/batch-generate` is called with `{ documentType, filter?, recordIds? }`
**THEN** the system creates a BullMQ job on the `pdf-batch-generate` queue
**AND** the job processes each record sequentially:
  1. Loads data context per record
  2. Compiles template + renders PDF per record
  3. Stores each PDF in the configured storage (S3/MinIO or local filesystem for MVP)
  4. Tracks progress: `{ total, completed, failed, errors[] }`
**AND** the endpoint returns immediately with `202 Accepted` and a `batchJobId`
**AND** batch progress can be queried via `GET /api/v1/documents/batch-generate/:batchJobId/status`
**AND** failed individual records don't abort the batch ŌĆö they are logged and skipped
**AND** batch size is limited to 500 records per request (returns 400 if exceeded)

### AC8: Template-Version Override Merging
**GIVEN** a `DocumentTemplateVersion` has override fields (`htmlOverride`, `cssOverride`, `headerOverride`, `footerOverride`)
**WHEN** a version is selected for rendering
**THEN** the override fields replace the base template's corresponding fields:
  - `htmlOverride` replaces `htmlTemplate` (if non-null)
  - `cssOverride` replaces `cssStyles` (if non-null)
  - `headerOverride` replaces `headerHtml` (if non-null)
  - `footerOverride` replaces `footerHtml` (if non-null)
**AND** email settings from the version (`emailSubject`, `emailBody`, `replyToEmail`, `ccEmails`) are available for the email pipeline (E12 does not implement email sending ŌĆö that's E10's responsibility, but the data must be accessible)
**AND** if no overrides are set on the version, the base template fields are used unchanged

  ```typescript
  function calculateMatchScore(version: DocumentTemplateVersion, context: VersionSelectionContext): number {
    let score = 0;
    if (version.languageCode && version.languageCode === context.languageCode) score += 10;
    if (version.branchCode && version.branchCode === context.branchCode) score += 8;
    if (version.numberSeriesId && version.numberSeriesId === context.numberSeriesId) score += 6;
    if (version.accessGroup && version.accessGroup === context.accessGroup) score += 4;
    if (version.customerGroupId && version.customerGroupId === context.customerGroupId) score += 2;
    // Penalise non-null mismatches
    if (version.languageCode && version.languageCode !== context.languageCode) score -= 20;
    if (version.branchCode && version.branchCode !== context.branchCode) score -= 16;
    return score;
  }
  ```
