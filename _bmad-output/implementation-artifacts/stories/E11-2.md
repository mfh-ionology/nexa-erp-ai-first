# Story 11.2: Task UI (Frontend)

Status: done

## Story

As a user,
I want to view, create, and manage tasks from a centralised "My Tasks" page and from embedded panels on every record detail page,
so that I can track action items across the entire ERP.

## Acceptance Criteria

1. GIVEN a user navigates to `/tasks` WHEN the page loads THEN the My Tasks page displays using the T1 Entity List template with status chip tabs (All/Open/In Progress/Overdue), search, priority filter, and task table ŌĆö matching `v0-nexa-design/app/(authenticated)/tasks/page.tsx`
2. GIVEN the task table WHEN a row is clicked THEN a Task Detail Sheet slides in from the right with full task info, status action buttons, assignee list, linked record, and activity timeline ŌĆö matching the `TaskDetailSheet` component in the v0 reference
3. GIVEN the user clicks "+ Create Task" WHEN the dialog opens THEN it shows title, description, priority, due date, assignees (multi-select), and optional linked record fields ŌĆö matching `CreateTaskDialog` in the v0 reference
4. GIVEN a record detail page (T2/T3) WHEN the Tasks tab/section renders THEN a `TaskPanel` component shows tasks linked to that entity with inline status actions ŌĆö matching `v0-nexa-design/components/tasks/task-panel.tsx`
5. GIVEN the Task Panel's "+ Add Task" button WHEN clicked THEN the Create Task Dialog opens with entityType and entityId pre-filled as a read-only linked record chip
6. GIVEN the briefing dashboard WHEN it loads THEN the "Tasks Today" card shows today's due + overdue tasks from the API with interactive status toggles ŌĆö matching `TasksCard` in `v0-nexa-design/components/dashboard/bottom-cards.tsx`
7. GIVEN batch selection on the My Tasks page WHEN tasks are selected THEN a batch action bar appears with "Complete All", "Reassign...", and "Cancel" actions
8. GIVEN a task status icon WHEN clicked THEN it cycles: Open ŌåÆ In Progress ŌåÆ Completed (matching the `cycleStatus` function in the v0 reference)
9. GIVEN any viewport WHEN the pages render THEN desktop shows full table, tablet hides Assignees column, phone shows card layout

### Backend API (E11.1 ŌĆö already built)

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/tasks` | List with filters (status, priority, entityType, entityId, assigneeId, search, limit, offset) |

| GET | `/tasks/:id` | Task detail with assignees |
| POST | `/tasks` | Create task (201) |
| PATCH | `/tasks/:id` | Update title/description/priority/dueDate |
| PATCH | `/tasks/:id/status` | Status transition (OPENŌåÆIN_PROGRESSŌåÆCOMPLETED; terminal states reject) |
| POST | `/tasks/:id/assignees` | Add assignee (201) |
| DELETE | `/tasks/:id/assignees/:userId` | Remove assignee (204) |
| DELETE | `/tasks/:id` | Soft delete (204) |

**Response shape**: `{ id, companyId, title, description, priority, status, dueDate, entityType, entityId, createdById, completedAt, deletedAt, createdAt, updatedAt, updatedBy, assignees: [{ id, userId, displayName, email }] }`
