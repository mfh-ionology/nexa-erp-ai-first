# Story E6.3: Screen Template System

Status: done

## Story

As a **developer**,
I want reusable page template components for all 8 screen types (T1-T8), so that every screen in the ERP follows a consistent layout without custom page designs — any new entity screen is built by selecting a template and populating it with module-specific configuration.

## Acceptance Criteria

1. GIVEN a developer building an entity list screen WHEN they use `<EntityListPage>` (T1) THEN it provides: breadcrumb, title, action buttons (`[+ New]`, `[AI]`, `[... More]`), saved view selector slot, search input, filter bar slot, data table with column sorting, row selection checkboxes, cursor-based pagination ("Load More"), and batch action bar that appears when rows are selected
2. GIVEN a developer building a record detail screen WHEN they use `<RecordDetailPage>` (T2) THEN it provides: breadcrumb, title with `<StatusBadge>`, action bar slot, tabbed content area, and related entities section
3. GIVEN a developer building a header+lines document WHEN they use `<HeaderLinesPage>` (T3) THEN it provides: breadcrumb, title with `<StatusBadge>`, action bar slot, tabbed header section, editable line items table with add/remove rows, auto-calculated totals section (subtotal, VAT, total), and `<EventFlowTracker>` slot
4. GIVEN template T4 (Briefing) WHEN `<BriefingPage>` renders THEN it provides a card-based grid layout for briefing items with action buttons, period comparison metrics, and configurable card types (KPI, action, alert)
5. GIVEN templates T5 (Board), T6 (Wizard), T7 (Settings), T8 (Report) WHEN they render THEN each provides the correct standardised layout per the UX Design Specification with appropriate action bar integration
6. GIVEN any template component WHEN rendered on mobile (<768px) THEN it adapts per the responsive behaviour matrix in the UX Design Spec (e.g., T1 uses cards instead of table rows, T3 header fields stack, T6 has full-width steps)
7. GIVEN any template component WHEN navigated via keyboard THEN it meets WCAG 2.1 AA requirements: semantic HTML landmarks, focusable interactive elements, ARIA labels on all controls
8. GIVEN all template components WHEN inspected THEN every user-facing string uses `t()` from `@nexa/i18n` — no hardcoded English text

    ```typescript
    import type { ColumnDef } from '@tanstack/react-table';

    /** Common props shared by all page templates */
    export interface BaseTemplateProps {
      /** Page title — should be a translation key passed through t() by the caller */
      title: string;
      /** Optional subtitle (e.g., record number) */
      subtitle?: string;
      /** Breadcrumb segments — each segment has label (translation key) and optional path */
      breadcrumbs: BreadcrumbSegment[];
      /** Whether the page is in a loading state */
      isLoading?: boolean;
      /** Children rendered in the template's main content area */
      children?: React.ReactNode;
    }

    export interface BreadcrumbSegment {
      label: string;
      path?: string;
    }

    /** T1: EntityListPage props */
    export interface EntityListPageProps<TData> extends BaseTemplateProps {
      /** Column definitions for TanStack Table */
      columns: ColumnDef<TData, unknown>[];
      /** Data rows */
      data: TData[];
      /** Entity type identifier (for query keys, i18n) */
      entityType: string;
      /** Whether more data is available (cursor pagination) */
      hasMore?: boolean;
      /** Callback to load next page */
      onLoadMore?: () => void;
      /** Whether next page is currently loading */
      isLoadingMore?: boolean;
      /** Search value (controlled) */
      searchValue?: string;
      /** Search change handler */
      onSearchChange?: (value: string) => void;
      /** Slot for filter bar content */
      filterSlot?: React.ReactNode;
      /** Slot for saved view selector */
      savedViewSlot?: React.ReactNode;
      /** Whether the user can create new records (controls [+ New] visibility) */
      canCreate?: boolean;
      /** Callback when [+ New] is clicked */
      onCreateNew?: () => void;
      /** Batch actions configuration — shown when rows are selected */
      batchActions?: BatchAction[];
      /** Row click handler (navigate to detail) */
      onRowClick?: (row: TData) => void;
      /** Unique row key extractor */
      getRowId?: (row: TData) => string;
      /** Overflow menu items for the list page [... More] */
      overflowActions?: OverflowAction[];
    }

    export interface BatchAction {
      key: string;
      labelKey: string;
      icon?: string;
      variant?: 'default' | 'destructive';
      onAction: (selectedIds: string[]) => void;
    }

    export interface OverflowAction {
      key: string;
      labelKey: string;
      icon?: string;
      onAction: () => void;
      section?: string;
    }

    /** T2: RecordDetailPage props */
    export interface RecordDetailPageProps extends BaseTemplateProps {
      /** Entity type identifier */
      entityType: string;
      /** Current entity status (for StatusBadge) */
      status?: string;
      /** Tab definitions */
      tabs: TabDefinition[];
      /** Active tab key (controlled) */
      activeTab?: string;
      /** Tab change handler */
      onTabChange?: (tabKey: string) => void;
      /** Slot for action bar component (rendered by E6.4) */
      actionBarSlot?: React.ReactNode;
      /** Related entities configuration */
      relatedEntities?: RelatedEntityConfig[];
      /** Slot for EventFlowTracker component (optional) */
      eventFlowSlot?: React.ReactNode;
    }

    export interface TabDefinition {
      key: string;
      labelKey: string;
      content: React.ReactNode;
      icon?: string;
    }

    export interface RelatedEntityConfig {
      key: string;
      labelKey: string;
      count: number;
      path?: string;
    }

    /** T3: HeaderLinesPage props */
    export interface HeaderLinesPageProps<TLine> extends BaseTemplateProps {
      /** Entity type identifier */
      entityType: string;
      /** Current entity status (for StatusBadge) */
      status?: string;
      /** Tab definitions for header section */
      headerTabs: TabDefinition[];
      /** Active header tab key (controlled) */
      activeHeaderTab?: string;
      /** Header tab change handler */
      onHeaderTabChange?: (tabKey: string) => void;
      /** Line item column definitions */
      lineColumns: ColumnDef<TLine, unknown>[];
      /** Line items data */
      lines: TLine[];
      /** Callback to add a new line */
      onAddLine?: () => void;
      /** Callback to remove a line by index */
      onRemoveLine?: (index: number) => void;
      /** Callback when a line cell is edited */
      onLineChange?: (index: number, field: string, value: unknown) => void;
      /** Whether lines are editable (false for read-only/approved docs) */
      isEditable?: boolean;
      /** Totals configuration */
      totals?: TotalsConfig;
      /** Slot for action bar component */
      actionBarSlot?: React.ReactNode;
      /** Slot for EventFlowTracker */
      eventFlowSlot?: React.ReactNode;
      /** Unique line key extractor */
      getLineId?: (line: TLine) => string;
    }

    export interface TotalsConfig {
      subtotal: string;
      vatAmount?: string;
      vatRate?: string;
      total: string;
      additionalLines?: { labelKey: string; value: string }[];
    }

    /** T4: BriefingPage props */
    export interface BriefingPageProps extends BaseTemplateProps {
      /** Briefing cards to display */
      cards: BriefingCardConfig[];
      /** Optional greeting message key */
      greetingKey?: string;
      /** User's first name for greeting */
      userName?: string;
      /** Optional summary/headline metrics */
      summaryMetrics?: SummaryMetric[];
    }

    export interface BriefingCardConfig {
      id: string;
      type: 'kpi' | 'action' | 'alert';
      titleKey: string;
      value?: string;
      previousValue?: string;
      format?: 'currency' | 'number' | 'percent';
      description?: string;
      actionLabelKey?: string;
      onAction?: () => void;
      severity?: 'info' | 'warning' | 'error';
      icon?: string;
    }

    export interface SummaryMetric {
      labelKey: string;
      value: string;
      previousValue?: string;
      format: 'currency' | 'number' | 'percent';
      trend?: 'up' | 'down' | 'neutral';
    }

    /** T5: BoardPage props */
    export interface BoardPageProps<TCard> extends BaseTemplateProps {
      /** Column definitions for the board */
      columns: BoardColumn<TCard>[];
      /** Callback when a card is moved between columns */
      onCardMove?: (cardId: string, fromColumn: string, toColumn: string) => void;
      /** Callback when a card is clicked */
      onCardClick?: (card: TCard) => void;
      /** Whether the user can create new cards */
      canCreate?: boolean;
      /** Callback when [+ New Card] is clicked */
      onCreateNew?: () => void;
      /** Slot for action bar */
      actionBarSlot?: React.ReactNode;
    }

    export interface BoardColumn<TCard> {
      key: string;
      labelKey: string;
      cards: TCard[];
      color?: string;
    }

    /** T6: WizardPage props */
    export interface WizardPageProps extends BaseTemplateProps {
      /** Step definitions */
      steps: WizardStep[];
      /** Current active step index (0-based) */
      activeStep: number;
      /** Callback to go to next step */
      onNext?: () => void;
      /** Callback to go to previous step */
      onBack?: () => void;
      /** Callback when wizard is completed */
      onComplete?: () => void;
      /** Whether the current step passes validation */
      isCurrentStepValid?: boolean;
      /** Slot for action bar */
      actionBarSlot?: React.ReactNode;
    }

    export interface WizardStep {
      key: string;
      labelKey: string;
      descriptionKey?: string;
      content: React.ReactNode;
      icon?: string;
      isOptional?: boolean;
    }

    /** T7: SettingsPage props */
    export interface SettingsPageProps extends BaseTemplateProps {
      /** Setting group definitions */
      groups: SettingsGroup[];
      /** Whether there are unsaved changes */
      isDirty?: boolean;
      /** Callback to save settings */
      onSave?: () => void;
      /** Callback to reset to defaults */
      onReset?: () => void;
      /** Slot for action bar */
      actionBarSlot?: React.ReactNode;
    }

    export interface SettingsGroup {
      key: string;
      labelKey: string;
      descriptionKey?: string;
      content: React.ReactNode;
      icon?: string;
      isCollapsible?: boolean;
      defaultOpen?: boolean;
    }

    /** T8: ReportPage props */
    export interface ReportPageProps<TResult> extends BaseTemplateProps {
      /** Parameter form content */
      parameterSlot: React.ReactNode;
      /** Whether the report has been run */
      hasResults?: boolean;
      /** Result column definitions */
      resultColumns?: ColumnDef<TResult, unknown>[];
      /** Result data rows */
      resultData?: TResult[];
      /** AI summary text slot */
      aiSummarySlot?: React.ReactNode;
      /** Callback to run the report */
      onRunReport?: () => void;
      /** Whether the report is currently running */
      isRunning?: boolean;
      /** Totals row data (optional) */
      totals?: Record<string, string>;
      /** Slot for action bar */
      actionBarSlot?: React.ReactNode;
    }
    ```

## Dev Notes

### Architecture Summary

This story creates the 8 standardised page templates that ALL screens in the ERP will use. No custom page layouts — developers select a template (T1-T8) and populate it with module-specific content via props. This is the single most important UX consistency mechanism in the application.

The templates are "headless" in the sense that they define layout, slots, and behaviour — but the specific data, columns, fields, and actions are provided by the module pages that consume them.

**Template usage counts across the ERP:**
| Template | Use Count | Examples |
|----------|-----------|---------|
| T1: Entity List | ~30 screens | Invoice List, Customer List, Item List |
| T2: Record Detail | ~30 screens | Customer Detail, Supplier Detail, Employee Detail |
| T3: Header+Lines | ~18 screens | Invoice, Sales Order, Purchase Order, Journal Entry |
| T4: Briefing | 1 screen | The Briefing (AI home screen) |
| T5: Board/Kanban | ~3 screens | CRM Pipeline, Production Schedule |
| T6: Wizard | ~5 screens | Company Setup, Payroll Run, Month-End Close |
| T7: Settings | ~12 screens | Company Settings, Module Settings, User Preferences |
| T8: Report | ~15 screens | Trial Balance, Aged Debtors, P&L, VAT Return |

### Template-ActionBar Integration

The ActionBar is built in E6.4 (the next story). Templates provide an `actionBarSlot` prop where the ActionBar will be rendered. For this story:
- `actionBarSlot` renders any `React.ReactNode` passed to it
- If no slot content is provided, the template renders its own minimal default actions (e.g., T1 renders `[+ New]` + `[⋯ More]` directly)
- E6.4 will provide the full `<ActionBar>` component that callers pass into the slot

### Data Table Architecture

The `<DataTable>` component wraps TanStack Table and is used by T1 (Entity List), T3 (Line Items), and T8 (Report Results). It provides:
- Column sorting (click headers)
- Row selection with checkboxes (T1 only)
- Cursor-based pagination via "Load More" pattern (not page numbers)
- Empty state with translated message
- Skeleton loading state
- Responsive column hiding (priority-based)

TanStack Table is headless — it provides the logic, we provide the rendering with Shadcn `<Table>` components.

### Responsive Design Strategy

Each template has three layout modes per the UX Design Spec responsive matrix:

| Template | Phone (<768px) | Tablet (768-1023px) | Desktop (>=1024px) |
|----------|---------------|---------------------|-------------------|
| T1: List | Cards instead of table | Table with priority columns | Full table with all columns |
| T2: Detail | Stacked fields, accordion tabs | Single-column form | Multi-column form + side panel |
| T3: Header+Lines | Stacked header, line cards | Single-column header, narrow table | Full layout |
| T4: Briefing | 1-column card grid | 2-column card grid | 3-column card grid |
| T5: Board | Single column, swipe | 3 columns visible | All columns |
| T6: Wizard | Progress bar + full-width content | Horizontal steps + content | Side step nav + content |
| T7: Settings | Stacked accordion | Single-column form | Two-column form |
| T8: Report | Collapsible params, card results | Params visible, table scroll | Full layout |

Use Tailwind responsive classes (`sm:`, `md:`, `lg:`) and the `useBreakpoint()` hook from E6.2 for programmatic breakpoint detection.

### Cross-Cutting Patterns (MANDATORY)

- **i18n**: Every user-facing string uses `t()` from `@nexa/i18n` — titles, button labels, column headers, empty states, ARIA labels, tooltips
- **Accessibility (NFR27, NFR28)**: Semantic HTML (`<main>`, `<nav>`, `<header>`, `<table>`), ARIA roles and labels, keyboard navigation, focus management, colour contrast >=4.5:1
- **TypeScript strict**: Generic type parameters for data-driven templates, no `any` types, proper discriminated unions for card/action types
- **Status colours**: Use CSS custom properties (`--color-status-*`) defined in E6.1's `globals.css`
