# Story E13b.5: Impersonation & Support Console

Status: done

## Story

As a **platform administrator**,
I want to impersonate a tenant for support purposes with mandatory reason, time limit, non-dismissable banner, and full action audit, so that I can troubleshoot issues while maintaining security and accountability.

## Acceptance Criteria

1. GIVEN the admin clicks "Impersonate" on a tenant WHEN a dialog appears THEN they must provide a text reason (mandatory) and the session has a configurable time limit (default 60 minutes)
2. GIVEN an impersonation session starts WHEN the admin is redirected to the tenant's ERP THEN a permanent non-dismissable amber banner shows: admin identity, tenant name, session expiry countdown, and "End Session" button
3. GIVEN an active impersonation session WHEN the session timer expires THEN the session auto-terminates and the admin is returned to the platform admin portal
4. GIVEN every action during impersonation WHEN it executes THEN it is logged in both the platform audit log and the tenant's audit log with `impersonatedBy` metadata
5. GIVEN the Support Console WHEN the admin searches THEN they can find tenants by domain, name, email, or ID

### Impersonation JWT Design

The impersonation JWT is distinct from the platform admin JWT and the tenant user JWT:

```typescript
// Platform admin JWT (existing)
{ sub: platformUserId, role: 'PLATFORM_ADMIN', type: 'platform' }

// Impersonation JWT (new — this story)
{ sub: platformUserId, tenantId: string, sessionId: string, type: 'impersonation', exp: expiresAt }

// Tenant user JWT (existing, ERP)
{ sub: userId, tenantId: string, companyId: string, type: 'user' }
```

The ERP API (`apps/api/`) auth middleware must be extended to recognise `type: 'impersonation'` JWTs and treat them as a super-user for that tenant while recording `impersonatedBy` metadata on all actions.

### Cross-Cutting Patterns (MANDATORY)

- **companyId**: NOT directly applicable to platform admin. However, the impersonation JWT needs to be recognized by the ERP API which uses companyId scoping — the impersonation user acts as a super-user across all companies in the tenant.
- **i18n**: NOT applicable. Platform Admin is English-only vendor tool. The ERP impersonation banner is also English-only (it's a platform admin tool, not a tenant user feature).
- **Audit**: Core feature of this story — dual audit logging (BR-PLT-015). Platform audit via existing `audit-on-response.hook.ts`. Tenant audit via ERP API middleware extension.
- **Visual Design**: Platform Admin pages follow dark sidebar theme (not Concept D). The impersonation banner in the ERP web app uses `bg-amber-500 text-black` for high-contrast safety indication.
- **Animations**: `animate-fade-in-up` for page transitions. Banner uses no animation (always visible, no transition).
