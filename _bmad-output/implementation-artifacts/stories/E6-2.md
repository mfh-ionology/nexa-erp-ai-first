# Story E6.2: Navigation Shell

Status: done

## Story

As a **user**,
I want a sidebar with module groups and company switcher, a top bar with user menu, notifications bell, and search, so that I can navigate the ERP efficiently across all modules and companies.

## Acceptance Criteria

1. GIVEN the authenticated user WHEN the app shell renders THEN a sidebar displays module groups (Finance, Sales, Purchasing, etc.) filtered by the user's enabled modules from resolved permissions
2. GIVEN the sidebar WHEN a module group is expanded THEN sub-items (entities within the module) are shown with Lucide icons and translated labels
3. GIVEN the sidebar WHEN the viewport is below 1024px THEN the sidebar collapses to an icon-only view with hover-to-expand behaviour; on mobile (<768px) it renders as an off-canvas drawer with hamburger toggle
4. GIVEN the top bar WHEN it renders THEN it shows: hamburger menu (mobile), company logo/name, unified search input placeholder, chat button placeholder, notifications bell placeholder (with unread count badge), and user avatar menu
5. GIVEN a multi-company tenant WHEN the user clicks the company switcher in the sidebar THEN a dropdown lists available companies and switching updates the company context globally (Zustand store + X-Company-Id header) and triggers a permissions re-fetch
6. GIVEN the user avatar menu WHEN clicked THEN it shows: user name, role, company name, "My Profile", "Preferences", "Sign Out" — all using translation keys
7. GIVEN the app shell WHEN rendered THEN it wraps the main content area with `<AppSidebar>`, `<AppHeader>`, and breadcrumb bar, and the main content resizes correctly when the sidebar collapses/expands
8. GIVEN any navigation element WHEN accessed via keyboard THEN it meets WCAG 2.1 AA requirements: Tab navigation, Escape to close dropdowns, Enter/Space to activate, ARIA landmarks for `<nav>`, `<main>`, `<header>`

## Dev Notes

### Architecture Summary

This story builds the visual navigation shell that wraps all authenticated pages. It establishes:

1. **Sidebar navigation**: Permission-filtered module groups with expand/collapse, company switcher, responsive collapse/icon-only/off-canvas modes
2. **Top header bar**: Search placeholder (wired in E6.5), chat toggle placeholder (wired in E6.5), notifications bell placeholder (wired in E9), user avatar dropdown
3. **App layout shell**: Semantic HTML with ARIA landmarks, skip link, breadcrumbs, responsive main content area
4. **Company switching**: Global company context change affecting all API calls and permissions

### Sidebar CSS Specifications

- Expanded width: 256px
- Collapsed width: 64px
- Background: `var(--color-surface)` (#ffffff)
- Border right: 1px solid `var(--sidebar-border)`
- Active item: bold text + 2px left purple border + `var(--color-background)` (#f4f2ff) background
- Hover: `var(--color-background)` (#f4f2ff) background
- Module group font: `var(--font-display)` (Plus Jakarta Sans), semi-bold
- Item font: `var(--font-body)` (Inter), regular
- Icon colour: `var(--color-text-muted)` (#6b7280), active: `var(--color-primary)` (#7c3aed)
- Transition: `width 200ms ease-out`

### Cross-Cutting Patterns (MANDATORY)

- **companyId**: Company switcher updates `activeCompanyId` in auth store. API client reads from store to send `X-Company-Id` header automatically.
- **i18n**: Every user-facing string uses `t()` from `@nexa/i18n` — sidebar labels, menu items, tooltips, ARIA labels
- **Permission filtering**: Sidebar only shows modules where user's permissions include `canAccess: true`. SUPER_ADMIN bypasses.
- **Accessibility**: WCAG 2.1 AA — semantic HTML, ARIA landmarks, keyboard navigation, focus management, skip link
