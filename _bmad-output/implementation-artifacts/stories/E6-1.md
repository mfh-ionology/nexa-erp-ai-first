# Story E6.1: React App Bootstrap

Status: done

## Story

As a **developer**,
I want a fully configured React application with Vite, React 19, TypeScript strict mode, TanStack Router, TanStack Query, Zustand stores, Tailwind CSS 4 + Shadcn UI, and JWT auth integration,
so that all subsequent frontend stories have a solid foundation with routing, state management, styling, and API communication.

## Acceptance Criteria

1. GIVEN the `apps/web` package WHEN it is built THEN it produces an optimised Vite + React 19 + TypeScript application with strict mode enabled
2. GIVEN the application loads WHEN the user is not authenticated THEN they are redirected to the login page
3. GIVEN the user authenticates WHEN the JWT is received THEN it is stored securely and used for all API requests via the shared API client
4. GIVEN the application WHEN React Query is configured THEN it provides caching, background refetch, and optimistic update capabilities for all server state
5. GIVEN Zustand stores WHEN client-side state is needed THEN stores exist for: auth state, sidebar state, active company, user preferences, and Co-Pilot drawer state
6. GIVEN the application WHEN the user interacts with forms THEN React Hook Form + Zod resolver is available for type-safe, schema-driven form validation
7. GIVEN the application WHEN any user-facing string is displayed THEN it uses the `@nexa/i18n` translation key system (`t()` function) — no hardcoded strings
8. GIVEN the application WHEN running in development THEN Vite HMR provides instant feedback with path aliases (`@/components`, `@/features`, etc.) resolved correctly

## Dev Notes

### Architecture Summary

This story transforms the stub `apps/web` package into a fully functional React application. It establishes:

1. **Build tooling**: Vite 6 + React 19 + TypeScript strict mode
2. **Styling**: Tailwind CSS 4 + Shadcn UI with purple theme tokens
3. **Routing**: TanStack Router with file-based routes, code splitting per module, auth/permission guards
4. **Server state**: TanStack Query with query key factory, global error handling, API client integration
5. **Client state**: Zustand stores for auth, sidebar, copilot, preferences
6. **Forms**: React Hook Form + Zod with i18n error messages
7. **i18n**: Full integration with `@nexa/i18n` — every string through `t()`
8. **Auth**: JWT token management with refresh interceptor, MFA support, secure storage

### Token Storage Security

Tokens are stored in-memory (Zustand store) — NOT in localStorage — to prevent XSS attacks from accessing them. The refresh token can optionally be stored in `sessionStorage` for session persistence. This follows the security pattern recommended for SPAs where httpOnly cookies are not feasible for the API architecture.

The API client intercepts 401 responses and attempts automatic token refresh before surfacing auth errors to the user.

### Routing Architecture

TanStack Router is used instead of React Router v7 (the Architecture doc mentions React Router v7 but TanStack Router is specified in the Epic). TanStack Router provides:
- File-based route generation (convention over configuration)
- Type-safe route params and search params
- Built-in code splitting support
- Route-level data loading via `beforeLoad`

The `_authenticated` layout route acts as the auth boundary — all routes under it require a valid JWT.
