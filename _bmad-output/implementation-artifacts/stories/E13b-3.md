# Story E13b.3: Billing Dashboard & Plan Management

Status: done

## Story

As a **platform administrator**,
I want an overview of payment status across all tenants with dunning level tracking and enforcement action controls, plus the ability to manage subscription plans and assign them to tenants,
so that I can manage billing health, take action on delinquent accounts, and control plan entitlements.

## Acceptance Criteria

1. GIVEN the Billing page WHEN it loads THEN a dashboard shows: total active tenants, payment status breakdown (current/grace/overdue/blocked), revenue summary, and enforcement action distribution
2. GIVEN a tenant with overdue billing WHEN the admin views the tenant's billing tab THEN they see dunning level (0-3), grace period remaining, last payment date, and enforcement action
3. GIVEN billing enforcement controls WHEN the admin changes enforcement from WARNING to READ_ONLY THEN the ERP webhook fires and the tenant enters read-only mode
4. GIVEN a plan change workflow WHEN the admin assigns a new plan THEN module entitlements and limits update immediately via webhook

## Dev Notes

### Architecture Context

This story delivers two frontend pages (**Billing Dashboard** and **Plans Management**) and enhances the per-tenant billing tab from E13b.2. It is **frontend-only** — all backend endpoints are already implemented in E3b.5:

| Backend Endpoint | Method | Purpose | Status |
|------------------|--------|---------|--------|
| `GET /admin/tenants` | GET | Tenant list (for billing aggregation) | Done (E3b) |
| `GET /admin/tenants/:id/billing` | GET | Per-tenant billing detail | Done (E3b.5 Task 3) |
| `PATCH /admin/tenants/:id/billing/enforcement` | PATCH | Change enforcement action | Done (E3b.5 Task 4) |
| `GET /admin/plans` | GET | List all plans | Done (E3b.5 Task 1) |
| `POST /admin/plans` | POST | Create plan | Done (E3b.5 Task 1) |
| `PATCH /admin/plans/:id` | PATCH | Update plan | Done (E3b.5 Task 1) |
| `POST /admin/tenants/:id/assign-plan` | POST | Assign plan to tenant | Done (E3b.5 Task 2) |

### Data Flow

```
Platform Admin UI  →  apiGet/apiPost/apiPatch  →  Platform API (port 5101)
     ↑                                                    ↓
React Query cache  ←  JSON response  ←  Prisma (Platform DB)
                                                          ↓
                                              BillingEnforcementService
                                                          ↓
                                              WebhookService → ERP tenant
                                                          ↓
                                              Entitlement cache bust (30s)
```

### Billing Enforcement State Machine (BR-PLT-004)

```
NONE ──→ WARNING ──→ READ_ONLY ──→ SUSPENDED
  ↑         ↓  ↑        ↓   ↑          ↓
  └─────────┘  └────────┘   └──────────┘
    (payment)   (override)   (full payment
                             + admin reactivate)
```

**Valid Transitions (from `billing-enforcement.service.ts`):**
| From | Allowed Targets |
|------|-----------------|
| NONE | WARNING |
| WARNING | NONE, READ_ONLY |
| READ_ONLY | NONE, WARNING, SUSPENDED |
| SUSPENDED | NONE |

Skipping levels is NOT allowed (e.g., NONE → READ_ONLY is invalid).

### Enforcement ↔ Billing Status Mapping

| Enforcement Action | Billing Status | ERP Impact |
|-------------------|----------------|------------|
| NONE | CURRENT | No restrictions |
| WARNING | GRACE | Warning banner in ERP |
| READ_ONLY | OVERDUE | All writes blocked (BR-PLT-005) |
| SUSPENDED | BLOCKED | Login blocked, data inaccessible |
