## Conflict Avoidance Rules

### Schema Conflicts
- **NEVER** run two epics that modify `packages/db/prisma/schema.prisma` simultaneously
- E13b is always safe to pair — it uses the Platform DB (`apps/platform-api/prisma/schema.prisma`)
- Between phases: always `git add . && git commit` and let migrations settle before starting the next pair

### Stagger Migrations
- If both epics in a phase add Prisma migrations, launch the first epic, wait for its migration story to complete, then launch the second
- Check migration status: `ls -la packages/db/prisma/migrations/ | tail -5`

### File Conflict Zones
These files are high-conflict — only one epic should touch them at a time:

| File | Risk |
|------|------|
| `packages/db/prisma/schema.prisma` | Schema migrations |
| `apps/api/src/index.ts` | Route registration |
| `apps/web/src/routeTree.gen.ts` | Auto-generated by TanStack Router |
| `apps/web/src/lib/navigation-config.ts` | Sidebar nav items |
| `packages/i18n/locales/en/*.json` | Translation keys |
| `pnpm-lock.yaml` | Dependency installs |

### Git Hygiene Between Epics
```bash
# After each epic completes:
gh auth switch --user mfh-ionology
git add .
git commit -m "feat(epic-EXX): <summary>"
git push origin main
```

Run this before launching each epic:
